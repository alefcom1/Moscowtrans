# Спецификация страниц разделов — Бюро переводов «Ремарка»

> **Назначение файла.** Техническое задание для генерации HTML-страниц разделов сайта moscowtrans.ru.  
> Каждый раздел — самостоятельная страница (не подраздел), от которой пойдут дочерние страницы услуг.  
> Все страницы создаются по единому шаблону `prototype/professionalnyj-perevod.html`.

---

## 1. Обязательные технические требования (общие для всех страниц)

### 1.1. Файлы и зависимости
- Расположение: `prototype/<slug>.html`
- CSS: `css/styles.css` (единый для всего сайта)
- JS: `js/main.js`, `js/hero.js`, `js/chat.js`, `js/animations.js` — все четыре в конце `<body>`
- Логотип: inline-скрипт переключения (как в `professionalnyj-perevod.html`, строки с `updateLogos()`)
- Favicon: `assets/favicon.ico`
- Шрифты: Inter + Sora с Google Fonts (уже подключены в шаблоне)

### 1.2. SEO-требования
- **`<title>`**: формат `{Ключевая фраза страницы} в Москве — {уточнение} | Ремарка`
- **`<meta name="description">`**: 140–155 символов, содержит главный ключевик, конкретный призыв
- **`<link rel="canonical">`**: `https://moscowtrans.ru/{slug}/`
- **`lang="ru"`** на теге `<html>`
- **H1**: первый и единственный на странице, содержит главный ключевик в именительном падеже
- **H2**: 4–7 штук, у одного из H2 — вариация главного ключевика с «в Москве»
- Ключевик в тексте: 5–8 употреблений в разных формах, без спама, органично

### 1.3. Микроразметка (JSON-LD в `<head>`)
Каждая страница содержит массив из трёх объектов:
1. `BreadcrumbList` — цепочка от главной
2. `Service` — описание услуги с provider (`@id: https://moscowtrans.ru/#organization`), areaServed Москва, offers с ценой
3. `FAQPage` — 4 вопроса из FAQ-блока страницы

### 1.4. Хлебные крошки (визуальные)
Внутри `.hero-bg-block`, перед `.hero-page`:
```
Главная › {Название раздела}
```
Ссылка «Главная» ведёт на `/`.

### 1.5. Структура страницы (порядок разделов)
1. `<header>` — шапка (копия из шаблона)
2. `.mobile-drawer` — мобильное меню
3. `.hero-bg-block` → хлебные крошки → `.hero-page` (чат + сайдбар)
4. `.sec.sec-intro` — H1 + подзаголовок-тэглайн + 4 фичи
5. `.sec.sec--alt.sec-docs` — аккордеон документов (секции специфичны для страницы)
6. `.sec.sec-translators` — «Наши переводчики»
7. `.sec.sec--alt.sec-how-pro` — «Как проходит работа» (6 шагов, идентично шаблону)
8. `.sec.sec-guarantees` — «Наши гарантии» (идентично шаблону)
9. `.sec.sec--alt.sec-vol-pricing` — таблица объём/стоимость (цена меняется по странице)
10. `.sec.sec-comparison` — таблица сравнения форматов (идентично шаблону)
11. `.sec.sec--alt.sec-reviews` — отзывы (3 карточки, релевантные теме)
12. `.sec.sec-faq` — FAQ (6–7 вопросов, последний — «Где заказать … в Москве?»)
13. `.sec.sec-cta` — призыв к действию
14. `#comments.comments-area` — блок комментариев WordPress
15. `.chat-float-btn` — плавающая кнопка чата
16. `<footer>` — подвал (копия из шаблона)
17. `<script>` — подключение JS + inline updateLogos()

### 1.6. Чат (блок Hero)
- Ольга, аватар `assets/olga.jpg`
- 3 приветственных сообщения с `data-greeting="1/2/3"` — уникальны для каждой страницы
- Первое сообщение содержит специфику страницы (вид перевода)
- Поле ввода, кнопки прикрепления файла и голосового ввода — идентично шаблону

### 1.7. Блок комментариев
- Структура: стандартный WordPress `#comments` / `.comments-area`
- CSS-классы: `.comment-list`, `.comment-body`, `.comment-author`, `.wp-avatar`, `.comment-respond`, `.comment-form` и т.д. — всё уже описано в `prototype/css/styles.css`
- 3–4 pre-filled комментария с именами и датами, тематически релевантными странице
- Форма: `#commentform` с полями comment, author, email, cookies-checkbox, submit
- Расположение: перед `<footer>`, после CTA

### 1.8. КРИТИЧЕСКИ ВАЖНО
- **Никогда не упоминать Краснодар** ни в каком видимом тексте
- Все тексты ориентированы на Москву
- Нотариальный перевод личных документов — **не продвигаем активно**
- «Личные документы» в аккордеоне — **не используем**
