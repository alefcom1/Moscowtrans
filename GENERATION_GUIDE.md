# Руководство по генерации страниц: Отрасли и Документы

## Контекст сайта

**Бюро переводов «Ремарка»**, Москва.  
Сайт — статический HTML-прототип с тёмной glassmorphism-темой (переключается через `data-theme="dark"/"light"` на `<html>`).  
Дизайн-система: CSS custom properties, `var(--bg-glass)`, `var(--border-subtle)`, `var(--r-lg)`.  
Шрифты: `--font-sans` (Inter) + `--font-serif` (Georgia).  
Цвет акцента: `#783CF0` (фиолетовый) → `#00A0F0` (синий).  
**Никогда не упоминать Краснодар** — только Москва и Россия.

---

## Типы страниц

### Тип A: Отраслевые страницы (`/otrasli/SLUG/`)
Описывают перевод в конкретной индустрии: медицине, IT, нефтегазе и т.д.  
Раздел: **«Отрасли»** в главном меню.

### Тип B: Страницы по документам (`/dokumenty/SLUG/`)
Описывают перевод конкретного вида документа: дипломов, договоров, справок и т.д.  
Раздел: **«Услуги → По типу документа»**.

---

## Структура HTML-страницы

Каждая страница создаётся по шаблону существующих сервисных страниц (`professionalnyj-perevod.html` как эталон).  
Подключаемые файлы: `css/styles.css`, `js/main.js`, `js/hero.js`, `js/chat.js`.

## Блок перелинковки (секция 12)

Добавляется **перед секцией отзывов** на каждой странице.  
Содержит: карточки 3–6 связанных страниц внутри раздела + кнопку-ссылку на главную страницу раздела.

## FAQ: Специфические вопросы по типам страниц

### Отраслевые страницы (3–4 вопроса дополнительно к общим)

- Есть ли у ваших переводчиков опыт работы в **[отрасли]**?
- Как вы обеспечиваете конфиденциальность технических данных?
- Переводите ли вы документы под тендерные требования?
- Работаете ли с международными стандартами (ISO, API, FIDIC и т.д.)?

### Страницы по документам (3–4 вопроса дополнительно)

- Нужно ли нотариальное заверение для использования за рубежом?
- Принимается ли ваш перевод в посольстве / суде / вузе?
- Сколько времени занимает апостилирование?
- Можно ли сделать перевод срочно — за 1 день?

## Schema.org JSON-LD

Каждая страница должна содержать:
- `BreadcrumbList` — цепочка навигации
- `LocalBusiness` — реквизиты бюро (копировать из существующих страниц, менять только `name`, `description`, `url`)
- `FAQPage` — все вопросы из секции FAQ

## Инструкция по генерации одной страницы

1. Выбрать тип (A — отрасль / B — документ) и SLUG из матриц выше.
2. Взять `professionalnyj-perevod.html` как основу.
3. Заменить все вхождения тематических текстов (H1, title, meta description, intro-text, doc-category-names).
4. В `sec-split` вставить заголовок и чеклист из соответствующей строки таблицы.
5. Добавить блок `sec-related` (секция 12) перед `sec-reviews` — заполнить карточками из колонки «Перелинковка с».
6. В `sec-docs` поменять категории документов на релевантные для данной отрасли/типа.
7. В `sec-translators` адаптировать описание переводчиков под специализацию.
8. В FAQ заменить/добавить отраслевые вопросы из раздела «Специфические вопросы» выше.
9. В хлебных крошках заменить ссылки и текущую страницу.
10. Обновить `<title>`, `<meta name="description">`, `<link rel="canonical">`, JSON-LD.
11. В блоке перелинковки текущую страницу **не включать** — только соседние.
